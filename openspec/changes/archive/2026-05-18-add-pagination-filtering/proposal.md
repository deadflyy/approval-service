## 为什么

当前系统中的列表页面（请示列表、模板列表、组织列表、用户列表、批次号列表等）采用一次性加载所有数据的方式展示。随着数据量增长，这会导致：
1. 页面加载缓慢，用户体验差
2. 前端内存占用过高
3. 无法快速定位特定数据

需要添加分页展示功能，支持每页数量选择、条件筛选和模糊查询，提升数据浏览效率。

## 变更内容

为所有列表页面添加：
- **分页组件**：支持翻页导航，显示当前页码和总页数
- **每页数量选择**：允许用户选择每页展示 10/20/50/100 条记录
- **条件筛选**：支持按状态、类型等字段进行精确筛选
- **模糊查询**：支持关键词搜索，在相关字段中进行模糊匹配

## 功能 (Capabilities)

### 新增功能
- `list-pagination`: 通用分页展示功能，包含分页组件、每页数量选择、筛选和搜索
- `api-pagination`: 后端 API 分页支持，修改现有接口支持分页参数

## 影响

- **前端组件**：Dashboard.vue, Requests.vue, Templates.vue, Organizations.vue, Users.vue, BatchNumbers.vue
- **后端 API**：所有返回列表数据的接口需要支持分页参数（page, pageSize, filters, keyword）
- **数据库查询**：需要优化查询，使用 LIMIT/OFFSET 进行分页
