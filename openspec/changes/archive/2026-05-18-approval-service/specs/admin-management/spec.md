## 新增需求

### 需求:用户管理
管理员必须能创建、编辑、删除用户，设置用户角色。

#### 场景:创建用户
- **当** 管理员填写用户名、密码、姓名、角色并提交
- **那么** 系统创建新用户，密码加密存储

#### 场景:编辑用户
- **当** 管理员修改用户信息
- **那么** 系统更新用户记录

#### 场景:删除用户
- **当** 管理员删除用户
- **那么** 系统标记用户为已删除（软删除）

### 需求:组织管理
管理员必须能维护党组织信息，包括组织名称、上级组织、组织层级。

#### 场景:创建组织
- **当** 管理员填写组织名称、上级组织、层级并提交
- **那么** 系统创建新组织记录

#### 场景:编辑组织
- **当** 管理员修改组织信息
- **那么** 系统更新组织记录

### 需求:模板管理
管理员必须能上传、编辑、删除模板文件，配置模板变量和映射关系。

#### 场景:上传模板
- **当** 管理员上传 docx 文件并配置类别、进度
- **那么** 系统保存模板文件，创建映射记录

#### 场景:编辑模板
- **当** 管理员修改模板的类别、进度映射
- **那么** 系统更新映射记录

### 需求:批复号管理
管理员必须能查看和修改当前批复号序列。

#### 场景:查看批复号
- **当** 管理员进入批复号管理页面
- **那么** 系统显示当前年份的批复号使用情况

#### 场景:重置批复号
- **当** 管理员重置批复号序列
- **那么** 系统将序号重置为指定值

### 需求:用户组织授权
管理员必须能为申请者配置可提交请示的组织列表。

#### 场景:配置授权
- **当** 管理员为用户选择可提交的组织
- **那么** 系统保存用户-组织授权关系

#### 场景:查看授权
- **当** 管理员查看用户详情
- **那么** 系统显示该用户被授权的组织列表
