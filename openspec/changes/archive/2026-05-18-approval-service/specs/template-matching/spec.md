## 新增需求

### 需求:模板配置
管理员必须能配置请示类别和进度到模板文件的映射关系。

#### 场景:新增模板映射
- **当** 管理员上传新的模板文件并配置类别为"换届"、进度为"换届-1"
- **那么** 系统保存模板文件，创建映射记录

#### 场景:查看模板列表
- **当** 管理员进入模板管理页面
- **那么** 系统显示所有模板及其类别、进度、变量列表

### 需求:变量提取
系统必须从模板文件中提取变量名（如 name、title、ref_org），供表单渲染使用。

#### 场景:解析模板变量
- **当** 管理员上传模板文件
- **那么** 系统自动解析模板中的变量，显示变量列表

### 需求:模板匹配
系统必须根据请示类别和进度值精确匹配对应的模板文件。

#### 场景:精确匹配
- **当** 请示类别为"增补"、进度为"增补-2"
- **那么** 系统匹配增补2模板

#### 场景:无匹配模板
- **当** 请示类别和进度没有对应模板
- **那么** 系统提示"未找到匹配的模板"，阻止生成批复

### 需求:批复文档生成
系统必须将请示数据填充到匹配的模板中，生成 Word(.docx) 格式的批复文档，保持模板原有的字体和行间距格式。

#### 场景:成功生成批复
- **当** 批复者审批通过请示
- **那么** 系统使用 docxtemplater 将请示数据填充到模板，生成可下载的 docx 文件

#### 场景:变量值为空
- **当** 模板中某个变量在请示数据中为空
- **那么** 系统使用空字符串替换，不显示占位符
