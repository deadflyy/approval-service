## 新增需求

### 需求:选择请示类别和进度
申请者提交请示时必须选择请示类别（成立/换届/增补/调整/更名/撤销/延期/架构调整）和进度值，系统根据这两个值匹配对应模板。

#### 场景:选择单步类别
- **当** 申请者选择类别为"更名"，进度为"更名"
- **那么** 系统匹配更名.docx模板，显示对应的表单字段

#### 场景:选择多步类别
- **当** 申请者选择类别为"换届"，进度为"换届-1"
- **那么** 系统匹配换届1模板，显示对应的表单字段

### 需求:动态表单渲染
系统必须根据选择的模板动态渲染表单字段，不同模板显示不同的字段组合。

#### 场景:换届类表单
- **当** 选择换届类别
- **那么** 表单必须包含：请示标题、请示页数、附件份数、涉及党组织名称、涉及党组织数、党员大会时间、书记候选人、副书记候选人、委员候选人、是否有高层担任班子、涉及支部数、涉及总支数、涉及党委、党委书记数、副书记数、委员数、总支书记数、副书记数、委员数、支部书记数、副书记数、委员数

#### 场景:更名类表单
- **当** 选择更名类别
- **那么** 表单必须包含：请示标题、请示页数、涉及党组织名称、新名称、涉及党组织数

#### 场景:撤销类表单
- **当** 选择撤销类别
- **那么** 表单必须包含：请示标题、请示页数、涉及党组织名称、涉及党组织数

### 需求:请示提交
申请者填写完表单后必须能提交请示，系统生成请示记录，状态为"待审批"。

#### 场景:成功提交
- **当** 申请者填写完所有必填字段并点击提交
- **那么** 系统创建请示记录，状态设为"待审批"，通知批复者

#### 场景:必填字段校验
- **当** 申请者未填写必填字段就提交
- **那么** 系统显示错误提示，阻止提交

### 需求:请示列表查看
申请者必须能查看自己提交的所有请示及其当前状态。

#### 场景:查看我的请示
- **当** 申请者进入"我的请示"页面
- **那么** 系统显示该用户提交的所有请示列表，包含标题、类别、进度、状态、提交时间
