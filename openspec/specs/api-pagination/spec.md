## ADDED Requirements

### 需求:API 分页参数支持
所有列表接口必须支持分页参数，使用 LIMIT/OFFSET 进行分页查询。

#### 场景:分页参数
- **当** 客户端请求列表接口
- **那么** 接口必须支持以下查询参数：
  - `page`：页码，整数，默认值 1，最小值 1
  - `pageSize`：每页数量，整数，默认值 20，可选值 10/20/50/100

#### 场景:分页参数验证
- **当** 客户端传递无效的分页参数
- **那么** 接口必须返回 400 错误
- **那么** 错误信息说明参数要求

### 需求:分页响应格式
所有列表接口必须返回统一的分页响应格式。

#### 场景:成功响应
- **当** 客户端请求列表接口成功
- **那么** 响应必须包含以下结构：
```json
{
  "data": [],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 100,
    "totalPages": 5
  }
}
```

#### 场景:空结果
- **当** 查询无匹配数据
- **那么** 返回空数组和正确的分页信息
- **那么** total 为 0，data 为空数组

### 需求:筛选参数支持
列表接口必须支持特定字段的筛选参数。

#### 场景:状态筛选
- **当** 客户端传递 `status` 参数
- **那么** 接口只返回该状态的记录
- **那么** 分页信息反映筛选后的总数

#### 场景:多条件筛选
- **当** 客户端同时传递多个筛选参数
- **那么** 接口使用 AND 逻辑组合筛选条件
- **那么** 分页信息反映所有条件筛选后的总数

### 需求:模糊查询支持
列表接口必须支持关键词模糊查询。

#### 场景:关键词搜索
- **当** 客户端传递 `keyword` 参数
- **那么** 接口在标题、名称等字段中进行 LIKE 查询
- **那么** 查询使用 `%keyword%` 模式匹配
- **那么** 分页信息反映搜索结果的总数

#### 场景:搜索字段
- **当** 不同接口定义不同的可搜索字段
- **那么** 每个接口必须明确列出可搜索的字段
- **那么** 搜索时使用 OR 逻辑组合多个字段

### 需求:排序支持
列表接口必须支持排序，默认按创建时间倒序。

#### 场景:默认排序
- **当** 客户端未指定排序参数
- **那么** 接口按 created_at DESC 排序

#### 场景:自定义排序
- **当** 客户端传递 `sortBy` 和 `sortOrder` 参数
- **那么** 接口按指定字段和方向排序
- **那么** 仅允许预定义的排序字段
