## 新增需求

### 需求:联络员请示列表
联络员必须能查看分配给自己的所有请示及其当前状态。

#### 场景:查看关联请示
- **当** 联络员进入"我的请示"页面
- **那么** 系统显示该联络员关联的所有请示，包含标题、类别、进度、状态、批复号

### 需求:更新流程状态
联络员必须能更新请示的流程状态：已签批、已打印、已领取。

#### 场景:标记已签批
- **当** 联络员点击"标记已签批"
- **那么** 系统将状态更新为"已签批"，记录签批时间

#### 场景:标记已打印
- **当** 联络员点击"标记已打印"
- **那么** 系统将状态更新为"已打印"，记录打印时间

#### 场景:标记已领取
- **当** 联络员点击"标记已领取"
- **那么** 系统将状态更新为"已领取"，记录领取时间

### 需求:分配联络员
管理员必须能为请示分配联络员。

#### 场景:分配联络员
- **当** 管理员在请示详情页选择联络员
- **那么** 系统将该请示关联到选定的联络员

#### 场景:联络员查看分配
- **当** 联络员登录系统
- **那么** 系统显示所有分配给该联络员的请示
