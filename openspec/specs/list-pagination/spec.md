## ADDED Requirements

### 需求:列表分页展示
系统必须为所有列表页面提供分页展示功能，使用服务端分页方式加载数据。

#### 场景:默认分页展示
- **当** 用户访问列表页面
- **那么** 系统默认展示第 1 页，每页 20 条记录
- **那么** 页面底部显示分页组件，包含页码导航和总记录数

#### 场景:翻页操作
- **当** 用户点击分页组件的页码或上下页按钮
- **那么** 系统加载对应页码的数据
- **那么** 页面滚动到顶部
- **那么** 分页组件高亮当前页码

#### 场景:切换每页数量
- **当** 用户在分页组件中选择每页展示数量（10/20/50/100）
- **那么** 系统重新加载第 1 页数据
- **那么** 分页组件更新总页数

### 需求:每页数量选择
系统必须允许用户选择每页展示的记录数量。

#### 场景:选择每页数量
- **当** 用户点击每页数量下拉选择器
- **那么** 显示选项：10、20、50、100
- **那么** 选择后立即生效并重新加载数据

#### 场景:记住每页数量选择
- **当** 用户选择每页数量后刷新页面
- **那么** 系统使用上次选择的每页数量（存储在 localStorage）

### 需求:条件筛选
系统必须支持按特定字段进行条件筛选。

#### 场景:状态筛选
- **当** 用户在筛选栏选择特定状态（如"待审批"）
- **那么** 系统只展示该状态的记录
- **那么** URL 参数更新为 `status=pending`
- **那么** 分页组件更新总记录数

#### 场景:清除筛选条件
- **当** 用户选择"全部"或清除筛选条件
- **那么** 系统展示所有记录
- **那么** URL 参数中移除对应的筛选参数

### 需求:模糊查询
系统必须支持关键词模糊查询，在相关字段中进行匹配。

#### 场景:关键词搜索
- **当** 用户在搜索框输入关键词
- **那么** 系统在标题、组织名称等字段中进行模糊匹配
- **那么** URL 参数更新为 `keyword=搜索词`
- **那么** 分页组件更新总记录数

#### 场景:清空搜索框
- **当** 用户清空搜索框
- **那么** 系统展示所有记录
- **那么** URL 参数中移除 keyword 参数

#### 场景:搜索防抖
- **当** 用户在搜索框输入关键词
- **那么** 系统等待 300ms 无输入后才发起搜索请求
- **那么** 避免频繁请求影响性能
