## 新增需求

### 需求:待审批列表
批复者必须能查看所有状态为"待审批"的请示列表。

#### 场景:查看待审批
- **当** 批复者进入待审批页面
- **那么** 系统显示所有待审批请示，包含标题、类别、进度、提交时间、申请者

### 需求:审批通过
批复者必须能审批通过请示，系统自动生成批复文档并分配批复号。

#### 场景:审批通过
- **当** 批复者点击"审批通过"
- **那么** 系统将状态更新为"已审批"，自动分配批复号，生成批复文档

#### 场景:批复号自动分配
- **当** 请示审批通过
- **那么** 系统查询当前年份最大批复号并加1，格式为"沪张江科综委〔2026〕X号"

### 需求:退回修改
批复者必须能将请示退回修改，申请者修改后可重新提交。

#### 场景:退回修改
- **当** 批复者点击"退回修改"并填写退回原因
- **那么** 系统将状态更新为"退回修改"，记录退回原因，通知申请者

#### 场景:重新提交
- **当** 申请者修改请示后点击"重新提交"
- **那么** 系统将状态更新为"待审批"，保留原请示记录

### 需求:状态流转
系统必须支持完整的审批状态流转：待审批→已审批→已签批→已打印→已领取。

#### 场景:状态流转
- **当** 联络员更新状态为"已签批"
- **那么** 系统记录签批时间，状态更新为"已签批"

#### 场景:状态回退
- **当** 批复者将已审批的请示退回修改
- **那么** 系统将状态更新为"退回修改"，撤销已分配的批复号

### 需求:批复文档下载
批复者和联络员必须能下载已生成的批复文档。

#### 场景:下载批复
- **当** 用户点击"下载批复"
- **那么** 系统返回对应的 docx 文件
